Estudo de caso e implementação de um sistema de compartilhamento de arquivos com base em IPFS e arquitetura Assíncrona.

O Objetivo do Projeto

O Aegis Share foi desenvolvido como um projeto acadêmico para explorar e validar a viabilidade de integrar protocolos de rede descentralizados (como o IPFS) com a arquitetura de um framework web tradicional e robusto como o Django. O foco principal é a resolução de desafios técnicos em torno da assincronicidade e da comunicação em tempo real.

Desafios Técnicos e Implementação

{% icon "cloud-off" class="text-info" style="width: 30px; height: 30px;" %}
IPFS: Integração P2P e Latência

A integração com o InterPlanetary File System (IPFS) foi o ponto central. O objetivo foi testar como um hash de conteúdo (CID) pode substituir um caminho de arquivo tradicional e como gerenciar operações de rede (que são lentas) sem travar o servidor web síncrono do Django.

Protocolos de Rede
{% icon "zap" class="text-danger" style="width: 30px; height: 30px;" %}
ASGI e Notificação Assíncrona

A implementação de Django Channels e Redis serviu como prova de conceito para o ambiente ASGI. Isso permitiu criar um fluxo de trabalho onde o upload para o IPFS é tratado de forma assíncrona, e o status do processamento é transmitido ao usuário instantaneamente via WebSockets.

Assincronicidade
{% icon "server" class="text-success" style="width: 30px; height: 30px;" %}
Django: Arquitetura e Controle

O Django (servido via Granian) estabelece a arquitetura MVC do projeto. Ele é responsável pelo gerenciamento de usuários, modelagem de dados e pelas URLs de acesso, funcionando como a ponte de controle entre a interface do usuário e o complexo subsistema IPFS/WebSockets.
