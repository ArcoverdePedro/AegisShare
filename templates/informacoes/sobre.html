Implementação de um sistema de compartilhamento de arquivos com base em IPFS e arquitetura Assíncrona.

O Objetivo do Projeto

O Aegis Share foi desenvolvido como um projeto acadêmico para explorar e validar a viabilidade de integrar protocolos de rede descentralizados (como o IPFS) com a arquitetura de um framework web tradicional e robusto como o Django. O foco principal é a resolução de desafios técnicos em torno da segurança e da imutabilidade de arquivos.

Escopo Técnico da Implementação

{% icon "lock" class="text-primary" style="width: 30px; height: 30px;" %}
Segurança

A integração com o InterPlanetary File System (IPFS) é um dos pilares do projeto. Em vez de salvar arquivos em um servidor central, eles recebem um hash (CID) e são distribuídos pela rede IPFS, o que garante sua imutabilidade. Senhas são protegidas com o algoritmo Argon2, padrão moderno de criptografia.

Velocidade e Assíncrona

Django Channels, Redis e a biblioteca htmx foram incorporados ao projeto com foco em garantir alta responsividade na interface. Essas tecnologias são operadas em conjunto para proporcionar atualizações em tempo real e respostas imediatas no front-end, oferecendo uma experiência de uso mais fluida e interativa.

Arquitetura Veloz

Estrutura

Servido via Granian, o Django estrutura o projeto com a arquitetura MVC de forma clara e robusta. Facilitando tanto a experiência do usuário quanto o trabalho dos Devs, ao centralizar o gerenciamento de usuários, modelagem de dados e rotas de acesso, atuando como elo eficiente entre a interface e o subsistema IPFS.
